# 인앱 사용 설명서 (❓ 도움말)

## 배경 / 문제

CutFlow는 기능이 많지만(파일 드롭 → 컷 기준 VAD/dB → Whisper 자막 →
FCPXML/SRT 내보내기 → AI 추천) 프로그램 안에서 사용법을 안내하는 수단이
없다. 처음 쓰는 사용자가 전체 흐름을 바로 파악하기 어렵다.

화면 어디서나 누를 수 있는 ❓ 버튼으로 **빠른 시작 가이드**를 보여준다.

## 목표

- 3단계 화면(설정/작업/결과) 어디서나 접근 가능한 ❓ 도움말 버튼.
- 클릭 시 **빠른 시작 가이드**(4단계 핵심 흐름)를 전용 다이얼로그로 표시.
- 내용은 간결하게 — 처음 쓰는 사람이 따라 할 수 있는 수준.

## 비목표 (YAGNI)

- 기능별 상세 매뉴얼(FEATURES.md 수준)을 앱에 담기.
- 다국어 도움말.
- 외부 파일/URL에서 도움말 로드(콘텐츠는 코드에 하드코딩).
- 도움말 내 검색/목차 네비게이션.

## 설계

### 1. 새 모듈 `ui/help_dialog.py`

도움말 다이얼로그와 콘텐츠를 한 파일에 격리한다 (main_window 비대화 방지,
CLAUDE.md 모듈 분리 패턴 준수).

- `HelpDialog(QDialog)` 클래스
  - 모달, 제목 "CutFlow 사용법", 크기 약 560×640
  - `QTextBrowser`(읽기 전용, 스크롤, HTML 렌더링)에 가이드 표시
  - 하단 "닫기" 버튼
- 콘텐츠는 모듈 상수 `_HELP_HTML`(HTML 문자열)로 하드코딩

### 2. 콘텐츠 (빠른 시작 가이드)

4단계 핵심 흐름 + 짧은 팁:

1. **파일 추가** — 영상을 화면에 드래그 앤 드롭(또는 `파일 선택`). 파일명
   순으로 자동 정렬.
2. **설정** — 컷 기준 선택(VAD 권장: 음성 구간만 유지), 최소 길이 조정,
   필요 시 `Whisper AI 자막 자동 생성` 체크.
3. **분석 시작** — 진행 상황과 타임라인이 실시간 표시. 일시정지/취소 가능.
4. **내보내기** — 이벤트명·프로젝트명 입력 후 FCPXML(필요 시 SRT) 저장 →
   Final Cut Pro로 바로 열기.

\+ 팁: VAD는 사람 목소리가 있는 구간만 남긴다 / 자막을 켜면 AI 제목·썸네일
추천도 받을 수 있다.

### 3. `ui/main_window.py` 연결

- `from ui.help_dialog import HelpDialog` 추가.
- `_step_header()`의 `h.addStretch()` **뒤에** ❓ 버튼 추가 → 세 화면 모두
  헤더 오른쪽에 표시(`_step_header`가 페이지마다 호출되므로 버튼도 각 페이지에
  독립 생성).
- 버튼 클릭 → `self._show_help()` → `HelpDialog(self).exec()`.

### 4. 문서 갱신

- `docs/FEATURES.md`: "인앱 도움말" 항목 추가.
- `docs/CHANGELOG.md`: 하위 호환 새 기능 → MINOR 항목 "추가"에 기록(버전
  번호는 릴리스 시점 합의).

## 데이터 흐름

```
❓ 버튼 클릭 (_step_header, 모든 화면)
  → MainWindow._show_help()
  → HelpDialog(self).exec()   # _HELP_HTML 표시, 모달
  → 닫기
```

## 테스트

- GUI 다이얼로그라 자동 테스트는 최소화.
- 가벼운 체크: `HelpDialog` import 가능 + `_HELP_HTML`이 비어있지 않은지.
- 실제 표시·레이아웃은 앱 실행 육안 확인.

## 주의사항

- Python 3.9 기준 — 유니온 타입 문법 금지.
- 이 기능은 FCPXML 이벤트/프로젝트명 기능과 별개. `main`에서 분기한
  `feature/in-app-help` 브랜치에서 작업.
