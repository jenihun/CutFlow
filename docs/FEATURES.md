# 기능 명세서

> 대상 프로그램: CutFlow  
> 대상 사용자: 액션캠으로 브이로그를 촬영하고 Final Cut Pro로 편집하는 개인 사용자  
> 최종 수정: 2026-06-15 (v0.9.0)

---

## 0. 작업 흐름 (3단계 화면)

`QStackedWidget` 기반 3단계 화면으로 진행하며, 단계별 버튼으로 이동한다.

1. **설정 화면** — 파일 추가, 컷 기준(VAD/dB), 무음·최소길이, Whisper 옵션, 백그라운드 모드 → `작업 시작 →`
2. **작업 화면** — 진행바·일시정지·취소 + 실시간 타임라인(컷 + 노란 자막 마커) + `[mm:ss] 자막내용` 로그가 **생성되는 즉시** 누적. 자막 클릭 시 타임라인 위치 강조 → `결과 보기 →`
3. **결과 화면** — 최종 타임라인(클릭 미리보기), FCPXML/SRT/자막복사 내보내기, AI 추천 → `← 처음으로`

- 하단에는 모든 화면 공용 상태바가 항상 표시된다.

## 1. 클립 입력

### 1-1. 드래그 앤 드롭
- 영상 파일을 앱 드롭존에 드래그하면 목록에 추가
- 중복 파일 자동 필터링

### 1-2. 파일 선택 다이얼로그
- 드롭존 내 `파일 선택` 버튼으로 파인더 열기
- 지원 확장자: `.mp4` `.mov` `.m4v` `.mts` `.m2ts` `.mkv` `.avi` `.insv`

### 1-3. 자동 정렬
- 파일명(날짜/시간 포함) 기준 오름차순 정렬
- 액션캠의 타임스탬프 파일명 규칙 전제 (예: `GX010001_20240115_143022.MP4`)

### 1-4. 목록 초기화
- `지우기` 버튼으로 전체 클립 제거

## 2. 컷 구간 자동 감지

`컷 기준` 드롭다운에서 두 방식 중 선택:

### 2-1. 음성 감지 (VAD · 기본 권장)
- Silero VAD(`faster_whisper.vad`)로 **사람 목소리가 있는 구간만** 유지 구간으로 잡음
- 엔진음·바람 등 큰 비발화 소음이 있어도 음성만 정확히 분리 — "자막 없는 곳 = 무음" 관점을 그대로 구현
- 말이 많아 음량 다이나믹 레인지가 좁은 영상에서도 정확
- `최소 길이`(이보다 짧은 침묵은 잇기) / 패딩(발화 앞뒤 여유, 최소 0.15초) 적용

### 2-2. 음량 기준 (dB)
- ffmpeg `silencedetect`로 설정 dB 이하를 무음으로 간주 (단순한 영상에 적합)

| 파라미터 | 기본값 | 범위 | 설명 |
|----------|--------|------|------|
| 무음 기준 | -30 dB | -60 ~ -10 dB | 낮을수록 더 민감하게 감지 (dB 모드 전용) |
| 최소 길이 | 0.5 초 | 0.1 ~ 3.0 초 | 이 길이 이상의 무음만 제거 (두 모드 공통) |

- 슬라이더 또는 옆 입력란에 **직접 수치 입력** 가능 (VAD 모드에서는 무음 슬라이더 비활성)

### 2-3. 자동 무음 기준 탐지 (🎯, dB 모드)
- 🎯 버튼으로 첫 클립의 첫 120초를 분석
- **VAD로 발화/비발화를 구분**한 뒤 각 구간의 RMS 분포 사이로 최적 dB를 계산
- 백그라운드 스레드(`NoiseDetectWorker`)에서 처리

### 2-4. 결과 표시
- 클립 목록에 구간 수, 제거 시간(초) 표시
- 예: `✅ 3구간 (12.4초 제거)`

## 3. 분석 제어

### 3-1. 분석 시작
- `분석 시작` 버튼 → 백그라운드 스레드에서 순차 처리
- 진행 바 + 현재 처리 중인 파일명 표시

### 3-2. 일시정지 / 재개
- 분석 중 `일시정지` 버튼 노출
- OS 시그널(SIGSTOP/SIGCONT)로 ffmpeg 프로세스를 실제로 정지 → 재개 시 처음부터 다시 하지 않음
- 클립 간 이동 시점에도 정지 가능

### 3-3. 작업 취소
- 분석 중 `취소` 버튼으로 즉시 중단 (`worker.stop()`)
- 취소 시 잠겼던 설정(슬라이더·입력란·CPU 모드) 자동 해제

### 3-4. 앱 종료 시 자동 정리
- 창 닫으면 진행 중인 ffmpeg 프로세스 자동 종료

## 4. Whisper AI 자막 생성

### 4-1. 활성화
- `Whisper AI 자막 자동 생성` 체크박스로 선택 활성화

### 4-2. 모델 선택

| 모델 | 속도 | 정확도 | 권장 용도 |
|------|------|--------|-----------|
| tiny | 매우 빠름 | 낮음 | 초안 확인 |
| base | 빠름 | 보통 | 빠른 작업 |
| small | 보통 | 높음 | — |
| medium | 느림 | 매우 높음 | — |
| large-v3 | 느림 | 최고 | 정확도 최우선 |
| large-v3-turbo | 보통 | 최고 수준 | **기본 권장** (최신) |

- `large-v3-turbo`: large-v3 수준 정확도, 속도 약 6배 빠름. 첫 실행 시 ~800MB 자동 다운로드
- `large-v3`: 첫 실행 시 ~1.5GB 자동 다운로드

### 4-3. 언어 선택
- 한국어 / English / 日本語 / 中文

### 4-4. 처리 엔진 (Apple Silicon 가속)
- **mlx-whisper** — M3 등 Apple Silicon에서 GPU/Neural Engine으로 추론. CPU 대비 대폭 빠름
- `mlx_whisper` 미설치 환경에서는 **faster-whisper(CPU)**로 자동 폴백
- 오디오를 Mac 로컬 임시 파일(16kHz 모노 WAV)로 추출 후 입력 → 외장 하드 연속 읽기 부담 최소화, 처리 후 자동 삭제

### 4-5. 실시간 스트리밍 + 정확도
- **30초 청크 단위 처리** — 클립 완료를 기다리지 않고 자막이 생성되는 즉시 작업 화면 로그·타임라인에 반영
- **정밀 타이밍** — `word_timestamps`로 실제 발화 경계에 맞춘 시각(싱크)
- **환각 필터** — 비발화음(배기음 등)에 붙는 반복 텍스트(`compression_ratio>2.4`), 깨진 문자(`�`), 구두점만 있는 줄 제거
- **자막 길이 분할** — 긴 자막을 단어 경계 기준 20자 이하 조각으로 나눠 가독성 확보(각 조각 정확한 시각)

### 4-6. 프레임 레이트 설정
- Whisper 옵션 내 FCPXML 출력 시퀀스 프레임 레이트 선택
- 옵션: 원본 유지 / 23.976 / 24 / 25 / 29.97 / 30 / 50 / 59.94 / 60
- "원본 유지" 선택 시 첫 번째 클립의 FPS를 그대로 사용

### 4-7. 타임라인 재계산
- Whisper가 생성한 자막의 시각이 원본 클립 기준이므로, 컷 이후 타임라인 기준으로 자동 재계산
- 제거된 구간에 해당하는 자막은 자동으로 제외

### 4-8. 결과 표시
- 클립 목록에 자막 줄 수 표시
- 예: `✅ 3구간 (12.4초 제거)  🗣 47줄`

## 5. 내보내기

### 5-1. FCPXML
- Final Cut Pro 호환 `.fcpxml` (버전 1.10) 생성
- 원본 영상을 복사하거나 재인코딩하지 않음 (편집 정보만 저장)
- **이름 없는 커스텀 포맷** — 비표준 해상도(DJI 2.7K 등)도 FCP가 거부하지 않음
- **프레임 정렬** — 모든 시간값(offset/duration/start)을 정수 프레임 배수로 스냅
- **내장 타임코드 반영** — DJI 등 촬영 시각 기반 TC를 읽어 asset/asset-clip `start` 기준점으로 사용 (없으면 0초 기준)
- **이벤트명 / 프로젝트명 지정** — 결과 화면 내보내기 영역에서 Final Cut의 이벤트명과 프로젝트(타임라인)명을 직접 입력. 프로젝트명 기본값은 오늘 날짜(`YYYY.MM.DD`)이며 수정 가능, 이벤트명은 필수 입력(빈 칸이면 내보내기 차단). 디스크 파일명과는 별개로 FCP 내부 표기를 결정
- `FCPXML 내보내기` 버튼으로 단독 출력. 저장 후 Final Cut Pro로 바로 열거나 파인더에서 위치 표시 선택 가능

### 5-1-1. FCP title 자막 삽입 (선택)
- Whisper 옵션의 `FCPXML에 자막 삽입 (Namsieon YT 템플릿)` 체크 시 활성
- Whisper 자막을 FCP **title 클립**으로 삽입 — `effect` 리소스(모션 템플릿 `.moti`)를 참조하고 asset-clip 내 `lane="1"`에 중첩
- title `offset`을 소스 타임코드 공간에 매핑해 컷 이후에도 정확히 동기
- 폰트·스타일은 **Namsieon YT 모션 템플릿**이 담당 (FCP에 설치 필요). SRT와 달리 폰트 지정 가능

### 5-2. SRT 자막 (Whisper 활성화 시)
- `SRT 내보내기` 버튼으로 단독 출력 (FCPXML과 분리)
- FCPXML과 동일 이름, `.srt` 확장자
- 예: `auto_cut.fcpxml` + `auto_cut.srt`
- Final Cut Pro에 드래그하여 자막 트랙으로 사용 가능

## 6. 성능 / CPU 제어

`백그라운드 모드 (저소음)` 체크박스 하나로 제어:

| 상태 | nice 레벨 | 설명 |
|------|-----------|------|
| 꺼짐 (기본) | 0 | 최대 속도, 발열/소음 증가 |
| 켜짐 | 15 | 팬 소음 최소, 처리 속도 느림 |

- ffmpeg `-threads 2` 제한으로 전체 코어 점유 방지
- 분석 중에는 슬라이더·입력란·CPU 설정이 잠기며, 완료/취소 시 해제

## 7. 타임라인 미리보기

작업 화면(실시간)과 결과 화면(최종) 양쪽에 타임라인이 표시된다.

### 7-1. 타임라인 시각화
- 클립별로 가로 막대 표시
- **초록**: 유지 구간 / **빨강**: 제거 구간 / **파란 점선**: 컷 포인트 / **노랑 레인**: 자막 위치
- 마우스 호버 → 주황 선 + 시간(초) 실시간 표시
- 작업 화면에서 자막 로그를 클릭하면 해당 자막 구간이 **주황 박스**로 강조

### 7-2. 미리보기 재생 (결과 화면)
- 타임라인 클릭 → 해당 시점 앞뒤 2.5초씩 총 5초를 ffplay로 재생
- ffplay 창이 별도로 열리며 재생 완료 후 자동 종료 (`-autoexit`)
- 재생 중인 파일명과 시점이 상태 표시줄에 표시

### 7-3. 스크롤
- 클립이 많을 경우 타임라인 영역이 스크롤됨

## 8. AI 제목 / 썸네일 추천

### 7-1. 활성화
- 메인 화면 하단 `🤖 AI 제목 / 썸네일 추천` 버튼으로 섹션 열기
- Whisper 자막 생성이 완료된 상태에서만 사용 가능

### 7-2. 지원 AI 제공자

| 제공자 | 모델 | 특징 |
|--------|------|------|
| Gemini | gemini-2.0-flash | Google API 키 필요, **무료 티어 지원** (`google-genai` SDK) |
| Claude | claude-sonnet-4-6 | Anthropic API 키 필요, 유료 |
| ChatGPT | gpt-4o-mini | OpenAI API 키 필요, 유료 |

- Gemini API 키 발급: https://aistudio.google.com → Get API key (무료)

### 7-3. API 키 관리
- 제공자별 API 키 입력 후 `저장` 버튼으로 로컬 저장 (`~/.cutflow/config.json`)
- 앱 재시작 시 자동으로 불러옴
- 제공자 전환 시 해당 키로 자동 전환

### 7-4. 결과 출력

**유튜브 제목 추천 (5개)**
- 클릭률을 고려한 매력적인 제목, 50자 이내
- 항목 클릭 → 클립보드 즉시 복사
- `전체 복사` 버튼으로 5개 한꺼번에 복사

**썸네일 문구 추천 (5개)**
- 썸네일 이미지에 들어갈 짧고 강렬한 문구, 15자 이내
- 각 항목 옆 `📋` 버튼으로 개별 복사

### 7-5. 처리 방식
- 자막 텍스트 최대 4,000자 추출해 AI에 전달
- 백그라운드 스레드에서 비동기 처리 (UI 블로킹 없음)

## 9. 자막 텍스트 복사

### 9-1. 기능
- 분석 완료 후 **자막 텍스트 복사** 버튼 활성화 (Whisper 자막이 있을 때만)
- 클릭 한 번으로 전체 클립의 자막 텍스트를 클립보드에 복사

### 9-2. 활용
- API 없이 AI 추천을 받고 싶을 때 사용
- 복사한 텍스트를 claude.ai / ChatGPT / Gemini 웹 채팅창에 붙여넣어 제목·썸네일 추천 요청 가능
- Claude Pro 구독자는 API 비용 없이 claude.ai에서 동일한 결과를 얻을 수 있음
